# 六幕場景規格（Scene Plan）— 正本

> 路線 A（Three.js 即時 3D）的敘事動線正本。由 Alvis 定調（2026-07-16）。
> 核心概念：**首尾呼應**——開頭只看到一張乾淨的工作桌；走完整段旅程後，
> 鏡頭回到同一張桌子，桌上已多出代表每段經歷的物件，才明白每個物件的意義。
>
> 敘事主軸：從工程師的桌面出發 → 進入大型系統 → 看見技術與帶領能力 →
> 飛向 AI 與個人產品 → 最後回到已累積完整經歷的自己。

### 2. 大型建築物（現有 city，造型優化）
- 沿用城市概念，但**不要灰土土的外型**——改成**紐約大都市的玻璃鏡面大樓**。
- **視覺參考：Marvel's Spider-Man 2 PS5（Insomniac）的曼哈頓**——寫實但風格化的玻璃摩天樓。
  - 技法：反射玻璃材質（envMap + 高 metalness / 低 roughness）+ 暖色窗光 emissive + bloom。
  - **紐約式階梯狀高低錯落天際線**（Art Deco setback，不要都同高的方塊）。
- **蜘蛛人彩蛋（已確認要埋）**：某棟樓側面一抹紅藍光，或天際線間一條若隱若現的蛛絲——懂的人才看得到，不張揚。
- ✅ 已做彩蛋：鋼架看板左上角有跨至背面的共生體液膜與觸手，右上角蛛網延伸至背架；材質會回應冷色月光。
- **鏡頭與互動重做（已確認 2026-07-16）**：
  - 遠景「大樓林立」establishing → 鏡頭**俯衝貼到一棟 hero 玻璃帷幕**。
  - 帷幕用 **`THREE.Reflector`（即時平面鏡）**，反射**對面街上一排發光霓虹招牌**——近距離 + 亮招牌
    是玻璃鏡面感最強的時機（遠看永遠不夠）。此幕由深夜開始接上暖黃破曉，仍保留暗玻璃與霓虹反射的對比。
  - **看板呈現（2026-07-17 更新）**：移除 `Knowledge Inbox / Workbench Nexus / Coming Soon` 三行霓虹字，改用 Alvis 提供的
    `MAKE COMPLEX THINGS FEEL SIMPLE.` 主視覺圖片鋪滿正面；圖片帶克制的自發光並映入 hero 玻璃。
- ✅ 已做：NASA 月面貼圖與高度圖、柔和月暈、低強度月光、密集階梯天際線、道路、窗燈、玻璃帷幕、屋頂鋼架看板與維修走道。
- ✅ 第二幕 caption 已移到左上方天空留白，與第一幕錯位並避開建築主輪廓。
- 對應：大型政府與企業系統經驗。

  轉譯為非武裝飛行載具。鏡頭採高空俯視城市與抽象航線，HUD 只呈現系統脈絡，不敘述巡檢、航拍或任務職責。

- 決策：無人機留為一幕；**knowledge-inbox 不另開場景**（它的卡片牆/知識圖會和第 5 幕 AI/RAG 撞臉），

### 4. 系統交付戰情室（新）— Digital systems command / war room
- **定調（已確認）：不是通用科幻機房，而是「你指揮大規模系統交付與監控的戰情室」。**
- 對應結尾主標三支柱的「**lead delivery（帶交付／技術帶領）**」：
  - 資料從第 2 幕的大樓「流進來」，中央一面狀態看板 / 系統網路圖，你在中間 orchestrate。
  - 表達：不只建系統，還能在大規模上營運、交付、帶團隊。
- 視覺：暗房 + 發光螢幕牆 / 中央看板 + 資料流動的光（bloom 讓螢幕發散）。
- **構圖（已確認 2026-07-17）**：中央指揮桌＋大型監控牆；顯示系統拓撲、服務健康度、部署進度與告警。
- **人物表達**：不製作人形；以中央座位、Alvis 名牌與操作中的控制台表示帶領角色。
- **保密規則**：真實系統名稱與資料不可公開，畫面一律使用合理的虛構服務、數值與代號。
- 避免與第 2 幕（系統）、第 5 幕（AI）撞臉：靠「有你在場指揮 + 監控看板」的敘事區隔，不做純抽象科幻。

### 5. AI × FigureShot 創意實驗室（新，視覺高潮，安排在倒數第二幕）
- 融合「AI 知識研究」與「FigureShot Lab」，做成**左右兩側對照**：
  - **右側（FigureShot Lab，見 figureshot-lab 專案）**：迷你攝影棚——公仔剪影（鋼彈/鋼鐵人/初音那種）、
    三腳架相機、聚光燈、發光的「靈感輪盤」。呼應其定位 Cinematic Editorial Workspace、
    首頁問句 "What do you want to shoot today?"。這是「把 AI 變成真產品」的一側。
  - **左側（AI / RAG / 檢索研究，含 knowledge-inbox）**：漂浮的論文、文件、檢索節點連成的知識網。
    knowledge-inbox（卡片牆 + 語意搜尋 + 知識圖）併在這裡表達。
- 表達重點：**不只研究 AI，還能把 AI 轉化成實際產品**。
- **已確認細節（2026-07-17）**：FigureShot 側採攝影棚、相機、燈光、構圖框與靈感輪盤；主體使用猛毒風格的流體剪影，
  不直接使用電影／漫畫圖片素材。加入 FigureShot 攝影 Instagram：`https://www.instagram.com/figsman99/`。
- AI 側以「文件進入 → Embedding → 檢索 → RAG 回答」為主流程；允許讀取 `figureshot-lab`、`knowledge-inbox`、
  `retrieval-comparison-thesis` 三個本機專案作為真實內容與視覺依據。
- 為什麼不用 AI 宇宙收尾：資訊量太大、看完不知下一步；適合當高潮不適合當終點。

## 全域：時間推移與氛圍（Alvis 2026-07-16）

- **背景不要太死太單調**：目前單一深藍太平。要依各場景做呼應調整，讓底色有層次。
- 第一幕已使用低速、低噪點 Color Bends；進入城市前淡回深夜天空，避免與月光／玻璃反射競爭。
- **時間從深夜慢慢到天亮**：整段旅程的光線/背景/fog 隨 progress 由深夜 → 金色破曉 → 清晨漸變
  （敘事對應：從摸黑起步 → 走到黎明、成為現在的自己）。這也一次解決「夜或日」的糾結：**兩者都要，是漸變**。
  - 做法：把靜態 dusk preset 換成「progress 驅動的時間系統」，插值 background/fog/燈光（之後連玻璃 env 也隨之亮）。
  - **時間線修正（2026-07-17）**：第二幕前、中段維持有月亮的完整深夜，只有城市離場前約 15% 開始接上暖黃破曉；
    第三至第五幕持續提亮，第六幕抵達完整早晨並把天空收斂為單色，避免球形天空在左下角露出色帶。
- **場景不早洩**：某一幕還沒結束前，不該看到下一幕（已用 fog far=60 讓相鄰島在飛過去前被藏住）。
